# C4 Architecture Document — Ayvalık Bank CC-1

The C4 model describes software architecture at four levels of zoom:

| Level | Diagram | Question answered |
|-------|---------|-------------------|
| 1 | **Context** | What does the system do and who uses it? |
| 2 | **Container** | What are the deployable units and technologies? |
| 3 | **Component** | What are the major building blocks inside each container? |
| 4 | **Code** | How are the key components implemented? |

> Note: The four levels are **Context, Container, Component, and Code** — the third level is *Component*, not *Class*.

---

## Level 1 — System Context

Who uses the system and what external systems does it interact with?

```mermaid
C4Context
    title System Context — Ayvalık Bank CC-1

    Person(admin, "Admin", "Bank administrator. Creates and deletes customers, sets transfer fees.")
    Person(customer, "Customer", "Bank customer. Opens accounts, deposits, withdraws, and transfers money.")

    System(ayvalikBank, "Ayvalık Bank CC-1", "Provides banking operations via a REST API. Manages customers, accounts, and transactions.")

    Rel(admin,    ayvalikBank, "Manages customers and settings", "HTTPS / Basic Auth")
    Rel(customer, ayvalikBank, "Manages accounts and money",     "HTTPS / Basic Auth")

    UpdateLayoutConfig($c4ShapeInRow="2", $c4BoundaryInRow="1")
```

**Key observations:**
- There are two types of users: **Admin** and **Customer**. Authentication is enforced via HTTP Basic Auth.
- There are no external third-party system integrations at this stage.
- The system is self-contained — all data lives in the bank's own database.

```mermaid
C4Container
```

**Containers and their roles:**

| Container | Technology | Responsibility |
|-----------|-----------|----------------|
| REST API Application | Java 21, Spring Boot 3.4, Spring Security, Spring Data JPA | All business logic, authentication, REST endpoints |
| Relational Database | PostgreSQL 16 | Durable storage for all domain data |

## Level 3 — Component

What are the major internal building blocks of the REST API application?

```mermaid
C4Component
    title Component Diagram — REST API Application

    Person(admin,    "Admin")
    Person(customer, "Customer")
    ContainerDb(postgres, "PostgreSQL", "PostgreSQL 16")

    Container_Boundary(app, "REST API Application") {

        Component(securityFilter, "Security Filter Chain",
            "Spring Security · Basic Auth",
            "Authenticates every request. Loads credentials from the customers table. Enforces ADMIN/CUSTOMER roles.")

        Component(adminCtrl, "AdminController",
            "Spring MVC · REST",
            "POST /api/admin/customers — create customer\nDELETE /api/admin/customers/{id}\nGET /api/admin/customers\nPUT /api/admin/settings/transfer-fee")

        Component(customerCtrl, "CustomerController",
            "Spring MVC · REST",
            "PUT /api/customers/{id}/password — change password")

        Component(accountCtrl, "AccountController",
            "Spring MVC · REST",
            "POST /api/accounts — open account\nGET /api/customers/{id}/accounts\nGET /api/accounts/{id}/balance\nPOST /api/accounts/{id}/deposit\nPOST /api/accounts/{id}/withdraw\nPOST /api/accounts/{id}/transfer\nGET /api/accounts/{id}/transactions")

        Component(customerAppSvc, "CustomerApplicationService",
            "Spring Service · @Transactional",
            "Implements: CreateCustomerUseCase, DeleteCustomerUseCase,\nListCustomersUseCase, ChangePasswordUseCase, SetTransferFeeUseCase.\nOrchestrates domain + infrastructure. Enforces password policy and reuse rules.")

        Component(accountAppSvc, "AccountApplicationService",
            "Spring Service · @Transactional",
            "Implements: CreateAccountUseCase, DepositMoneyUseCase,\nWithdrawMoneyUseCase, GetBalanceUseCase, GetTransactionsUseCase,\nTransferMoneyUseCase, ListAccountsUseCase.\nOrchestrates domain + infrastructure. Applies transfer fees.")

        Component(domainLayer, "Domain Layer",
            "Pure Java — zero framework imports",
            "Entities: Customer, Account, Transaction.\nValue Objects: Money, Password, *Id records.\nDomain Services: PasswordValidationService, TransferDomainService.\nPorts In: 12 use-case interfaces.\nPorts Out: 5 infrastructure interfaces.")

        Component(persistenceAdapters, "Persistence Adapters",
            "Spring Data JPA · Hibernate",
            "CustomerPersistenceAdapter → CustomerRepositoryPort\nAccountPersistenceAdapter → AccountRepositoryPort\nTransactionPersistenceAdapter → TransactionRepositoryPort\nSettingsPersistenceAdapter → SettingsRepositoryPort\nMaps JPA entities ↔ domain objects.")

        Component(securityAdapter, "BCryptPasswordHasherAdapter",
            "Spring Security · BCrypt (strength 12)",
            "Implements PasswordHasherPort.\nProvides hash() and matches() without exposing BCrypt to the domain.")
    }

    Rel(admin,    securityFilter, "HTTP request", "Basic Auth")
    Rel(customer, securityFilter, "HTTP request", "Basic Auth")

    Rel(securityFilter, adminCtrl,    "Forwards if ROLE_ADMIN")
    Rel(securityFilter, customerCtrl, "Forwards if ROLE_CUSTOMER")
    Rel(securityFilter, accountCtrl,  "Forwards if ROLE_CUSTOMER")

    Rel(adminCtrl,    customerAppSvc, "Calls use cases")
    Rel(customerCtrl, customerAppSvc, "Calls use cases")
    Rel(accountCtrl,  accountAppSvc,  "Calls use cases")

    Rel(customerAppSvc, domainLayer,        "Calls domain entities and services")
    Rel(accountAppSvc,  domainLayer,        "Calls domain entities and services")
    Rel(customerAppSvc, persistenceAdapters,"Calls via outbound ports")
    Rel(accountAppSvc,  persistenceAdapters,"Calls via outbound ports")
    Rel(customerAppSvc, securityAdapter,    "Calls PasswordHasherPort")

    Rel(persistenceAdapters, postgres, "Reads / Writes", "JPA / JDBC")

    UpdateLayoutConfig($c4ShapeInRow="3", $c4BoundaryInRow="1")
```

**Component responsibilities:**

| Component | Pattern | Notes |
|-----------|---------|-------|
| `AdminController` | Inbound Adapter | Translates HTTP → `Command` records → use-case interfaces |
| `CustomerController` | Inbound Adapter | Password change endpoint |
| `AccountController` | Inbound Adapter | All account-related endpoints |
| `CustomerApplicationService` | Application Service | No business rules — orchestration only |
| `AccountApplicationService` | Application Service | No business rules — orchestration only |
| Domain Layer | Domain (Hexagonal core) | Pure Java, all invariants and business rules |
| `*PersistenceAdapter` | Outbound Adapter | Implements domain repository ports using JPA |
| `BCryptPasswordHasherAdapter` | Outbound Adapter | Implements `PasswordHasherPort` using Spring Security |
| Security Filter Chain | Cross-cutting | Basic Auth, role enforcement, `BankUserDetailsService` |

## Level 4 — Code

Detailed class-level view for two key flows.

### 4a. Transfer Money — Class Interaction

```mermaid
classDiagram
    direction LR

    class AccountController {
        <<inbound adapter>>
        +transfer(accountId, TransferRequest) ResponseEntity
    }

    class TransferMoneyUseCase {
        <<port in>>
        +transfer(Command) void
    }

    class TransferMoneyCommand {
        <<record>>
        +AccountId sourceAccountId
        +AccountId targetAccountId
        +Money amount
    }

    class AccountApplicationService {
        <<application service>>
        +transfer(Command) void
    }

    class Account {
        <<entity>>
        +transferOut(amount, fee, targetId) Transaction
        +transferIn(amount, sourceId) Transaction
    }

    class TransferDomainService {
        <<domain service>>
        +calculateFee(amount, sameCustomer, feePercent) Money
    }

    class AccountRepositoryPort {
        <<port out>>
        +findById(AccountId) Optional~Account~
        +save(Account) Account
    }

    class TransactionRepositoryPort {
        <<port out>>
        +save(Transaction) Transaction
    }

    class SettingsRepositoryPort {
        <<port out>>
        +getTransferFeePercent() BigDecimal
    }

    class AccountPersistenceAdapter {
        <<outbound adapter>>
        +findById(AccountId) Optional~Account~
        +save(Account) Account
    }

    AccountController      --> TransferMoneyUseCase       : calls
    TransferMoneyUseCase   <|.. AccountApplicationService : implements
    TransferMoneyUseCase   +-- TransferMoneyCommand
    AccountApplicationService --> Account                 : deposit / transferOut
    AccountApplicationService --> TransferDomainService   : calculateFee()
    AccountApplicationService --> AccountRepositoryPort   : find / save
    AccountApplicationService --> TransactionRepositoryPort : save
    AccountApplicationService --> SettingsRepositoryPort  : getTransferFeePercent()
    AccountRepositoryPort  <|.. AccountPersistenceAdapter : implements
```

### 4b. Change Password — Class Interaction

```mermaid
classDiagram
    direction LR

    class CustomerController {
        <<inbound adapter>>
        +changePassword(customerId, ChangePasswordRequest) ResponseEntity
    }

    class ChangePasswordUseCase {
        <<port in>>
        +changePassword(Command) void
    }

    class ChangePasswordCommand {
        <<record>>
        +CustomerId customerId
        +String rawNewPassword
    }

    class CustomerApplicationService {
        <<application service>>
        +changePassword(Command) void
    }

    class PasswordValidationService {
        <<domain service>>
        +validate(String rawPassword) void
    }

    class Customer {
        <<entity>>
        +changePassword(Password) void
        +getAllPasswordsForReuseCheck() List~Password~
    }

    class PasswordHasherPort {
        <<port out>>
        +hash(String) String
        +matches(String, String) boolean
    }

    class CustomerRepositoryPort {
        <<port out>>
        +findById(CustomerId) Optional~Customer~
        +save(Customer) Customer
    }

    class BCryptPasswordHasherAdapter {
        <<outbound adapter>>
        +hash(String) String
        +matches(String, String) boolean
    }

    CustomerController          --> ChangePasswordUseCase          : calls
    ChangePasswordUseCase       <|.. CustomerApplicationService    : implements
    ChangePasswordUseCase       +-- ChangePasswordCommand
    CustomerApplicationService  --> PasswordValidationService      : validate format
    CustomerApplicationService  --> CustomerRepositoryPort         : findById / save
    CustomerApplicationService  --> PasswordHasherPort             : matches / hash
    CustomerApplicationService  --> Customer                       : changePassword()
    PasswordHasherPort          <|.. BCryptPasswordHasherAdapter   : implements
```

## Dependency Flow Summary

All dependencies point inward — the domain layer has no knowledge of any outer layer:

```
HTTP Request
    │
    ▼
[Spring Security Filter] ── authenticates via BankUserDetailsService
    │
    ▼
[Inbound Adapter] ── REST Controller
    │  builds Command record
    ▼
[Port In] ── Use Case Interface  (domain)
    │
    ▼
[Application Service] ── @Transactional orchestration
    │                          │
    ▼                          ▼
[Domain Layer]           [Port Out] ── Repository / Hasher Interface  (domain)
Entities · Services           │
Value Objects                 ▼
                        [Outbound Adapter]
                        JPA Persistence · BCrypt
                              │
                              ▼
                        [PostgreSQL Database]
```
